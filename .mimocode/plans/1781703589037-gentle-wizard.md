# Fix Timer / Sync Button Issues on Admin Page

## Problem Analysis

### "Configuration globale" section
- **"Forcer la synchronisation" button is a no-op**: `forceSync()` in `use-live-data.tsx:57` only sets `lastSync = Date.now()` — no API call, no backend sync triggered
- **"Dernière sync" shows a fake timer tick**, not actual last sync time
- **"Fréquence de rafraîchissement" input** controls a `setInterval` that just increments `lastSync` — disconnected from actual data refresh

### "Supervision des flux API" section  
- **Pipeline status auto-refresh (15s) and audit logs (10s) work correctly** ✅
- **"Dernière sync" timer updates live** via the `now` 1-second tick ✅
- **Pipeline Exécuter polling** has no unmount cleanup (minor)

### Backend scheduler is working correctly
- `console.php` has 5 scheduled commands using `SyncSetting` intervals via `isSyncDue()` ✅
- "Configuration Sync Backend" config UI works correctly ✅

## Changes

### 1. `app/Http/Controllers/AdminController.php` — Add `triggerSyncAll()`
New method: triggers all 5 sync commands (`sync:quality`, `sync:production`, `sync:logistics`, `sync:drive`, `sync:gpro`) via `Process::run()`, creates audit log, returns JSON.

### 2. `routes/web.php` — Add route
```
Route::post('/pipeline/sync-all', [AdminController::class, 'triggerSyncAll']);
```

### 3. `resources/js/services/adminApi.ts` — Add `triggerAllSync()`
Export new function calling `POST /admin/pipeline/sync-all`.

### 4. `resources/js/hooks/use-live-data.tsx` — Remove fake timer
- Remove the `setInterval` that auto-increments `lastSync` (lines 41-47)
- `lastSync` now only updates when `forceSync()` is called manually
- Keep `now` 1-second tick (needed for "il y a X min Xs" displays)
- Keep `refreshIntervalSec` / `setRefreshIntervalSec` in context (used by admin page to control auto-refresh intervals)

### 5. `resources/js/pages/admin.tsx` — Three fixes

**a) "Forcer la synchronisation" button (lines 560-568):**
- Import `triggerAllSync` from `adminApi.ts`
- On click: call `await triggerAllSync()`, then `forceSync()` to update timestamp, then refresh pipeline status + audit logs
- Add disabled/loading state while sync is in progress

**b) Use `refreshIntervalSec` to control auto-refresh intervals (lines 287-310):**
- Replace hardcoded `10000` (audit logs) and `15000` (pipeline) with `refreshIntervalSec * 1000`
- This makes the "Fréquence de rafraîchissement" input actually control data refresh rate
- Add `refreshIntervalSec` to the useEffect dependency array (restart intervals when value changes)

**c) Pipeline Exécuter polling cleanup (lines 865-889):**
- Use `useRef` to track timeout IDs and clear on unmount
- Add `isMounted` guard inside `poll` callback
