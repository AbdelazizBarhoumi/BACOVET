# Plan: Store All Synced Data Historically

## Context
Every sync command (`sync:quality`, `sync:production`, `sync:logistics`, `sync:drive`, `sync:gpro`) pulls raw data from external APIs into local DB tables. Most tables get **TRUNCATED** on each sync (previous data is wiped). Some tables use **UPSERT** (history preserved naturally). The goal is to store **all raw synced data** after every sync so we have a complete audit trail.

## Proposed Solution

### 1. New Migration: `data_snapshots` table
A single unified table using JSON to store raw synced data per table per sync batch:

```php
Schema::create('data_snapshots', function (Blueprint $table) {
    $table->id();
    $table->string('source');           // 'novacity_quality', 'novacity_production', 'novacity_logistics', 'google_drive', 'gpro_consulting'
    $table->string('table_name');       // e.g. 'check_pass_qte', 'vue_stock'
    $table->timestamp('synced_at');     // when the sync happened
    $table->integer('row_count');       // number of rows captured
    $table->json('data');               // the actual rows as JSON array
    $table->timestamps();
    $table->index(['source', 'table_name']);
    $table->index('synced_at');
    $table->index('table_name');
});
```

### 2. New Service: `DataSnapshotService`
- `snapshotTable(string $source, string $tableName)` — reads all rows from the table and stores as JSON in `data_snapshots`
- `snapshotTables(string $source, array $tableNames)` — batch snapshot multiple tables
- Called at the END of each sync method in `SyncService`, after all data is loaded
- Only snapshots tables that exist and have data

### 3. Modify `SyncService` — Add snapshot calls
After each sync method completes, call `DataSnapshotService::snapshotTables()`:

- `syncQuality()` → snapshot all 10 quality tables
- `syncProduction()` → snapshot all 18 production tables
- `syncLogistics()` → snapshot all 16 logistics tables
- `syncGoogleDrive()` → snapshot all 9 Google Drive tables
- `syncGproConsulting()` → snapshot all 4 GPRO tables

### 4. New API Endpoint: `GET /data-snapshots/{tableName}`
- Returns historical snapshots for a given table name
- Query params: `from`, `to`, `limit` (default 30)
- Each result includes: `synced_at`, `row_count`, `data` (JSON)
- Optional: `GET /data-snapshots/{tableName}/{snapshotId}` for a specific snapshot

### 6. Implementation Details

**DataSnapshotService** pseudocode:
```php
class DataSnapshotService {
    public function snapshotTables(string $source, array $tables): void {
        foreach ($tables as $tableName) {
            if (!Schema::hasTable($tableName)) continue;
            $rows = DB::table($tableName)->get();
            if ($rows->isEmpty()) continue;
            DB::table('data_snapshots')->insert([
                'source' => $tableName, // store table name as source for easy querying
                'table_name' => $tableName,
                'synced_at' => now(),
                'row_count' => $rows->count(),
                'data' => $rows->toJson(),
            ]);
        }
    }
}
```

**SyncService** modification (example for syncQuality):
```php
public function syncQuality(): void {
    // ... existing sync code ...
    $this->snapshot->snapshotTables('novacity_quality', [
        'check_pass_qte', 'vw_defects', 'qcm_defect_trx',
        'pieces_ok_jour', 'pieces_produites_jour',
        'pieces_ok_annee', 'pieces_produites_annee',
        'rejets_inspection_paquet',
    ]);
}
```
