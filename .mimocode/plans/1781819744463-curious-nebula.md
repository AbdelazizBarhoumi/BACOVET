# Plan: Fix LIVE SYNC Indicator Behavior

## Problem Summary

The `LiveSyncPill` indicator has several critical issues:

1. **Disconnected from actual data fetches**: Each page fetches data every 60s (hardcoded), but the pill doesn't know about it. `lastSync` in the context is only updated by `forceSync()` (clicking the pill).

2. **Admin refresh interval ignored**: The "Fr├Еquence de rafra├«chissement" setting updates `refreshIntervalSec` in context, but pages hardcode `setInterval(fetchData, 60000)` instead of using this value.

3. **Missing orange state**: Spec requires three colors: green (<2min), orange (2-10min), red (>10min or error). Currently only green/red.

4. **No error tracking**: Pill doesn't reflect when API fetches fail.

## Root Cause

The `LiveDataProvider` context and page-level fetch logic are completely disconnected:
- Context: `lastSync` only updates on `forceSync()` click
- Pages: Each has local `lastSync` state, updates it after fetch, never informs context
- Result: Pill shows "LIVE SYNC: OK" based on a timer that started at mount, not actual data freshness

## Solution

### 1. Enhance `LiveDataProvider` with fetch tracking

**File: `resources/js/hooks/use-live-data.tsx`**

Add methods for pages to report fetch events:
```typescript
type Ctx = {
  lastSync: number;
  now: number;
  isStale: boolean;
  refreshIntervalSec: number;
  setRefreshIntervalSec: (n: number) => void;
  forceSync: () => void;
  recordFetchSuccess: () => void;  // NEW
  recordFetchError: () => void;    // NEW
  hasError: boolean;               // NEW
};
```

- `recordFetchSuccess()`: Sets `lastSync = Date.now()`, clears error flag
- `recordFetchError()`: Sets `hasError = true`
- `hasError`: Boolean flag, cleared on next success or `forceSync()`

### 2. Update `LiveSyncPill` with three-state colors

**File: `resources/js/components/LiveSyncPill.tsx`**

Color logic per spec:
- **Green**: `hasError === false && (now - lastSync) < 120_000` Рєњ "LIVE SYNC: OK"
- **Orange**: `hasError === false && (now - lastSync) >= 120_000 && (now - lastSync) < 600_000` Рєњ "SYNC: ATTENTION"
- **Red**: `hasError === true || (now - lastSync) >= 600_000` Рєњ "SYNC: ERREUR"

Remove `lastFetchTime` and `hasError` props Рђћ use context only.

### 3. Connect pages to context

**Files: All page components**

For each page (quality, production, logistics, development, methods):

1. Import `useLiveData` hook
2. Replace hardcoded `setInterval(fetchData, 60000)` with `refreshIntervalSec * 1000`
3. After successful fetch: call `recordFetchSuccess()`
4. After failed fetch: call `recordFetchError()`
5. Remove local `lastSync` state and "Derni├еre sync" display (pill shows this)

### 4. Persist refresh interval to backend (optional)

**File: `app/Http/Controllers/Api/AdminController.php`**

The refresh interval is currently client-only (React state). To persist across sessions:
- Add `refresh_interval` to `sync_settings` table
- Load on app init, save on admin change
- This is a nice-to-have, not required for core fix
