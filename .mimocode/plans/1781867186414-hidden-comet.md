# Plan: Per-Source API Sync Toggles

## Goal
Add `.env` switches for each external API source so sync jobs can be disabled per-source without removing them from the scheduler. When a source is disabled, the scheduler skips it entirely (no API calls, no DB writes).

## Sources to Toggle
| Source | Scheduler Command | Env Variable | Default |
|--------|------------------|--------------|---------|
| Novacity (Quality) | `sync:quality` | `SYNC_NOVACITY_QUALITY_ENABLED` | `true` |
| Novacity (Production) | `sync:production` | `SYNC_NOVACITY_PRODUCTION_ENABLED` | `true` |
| Novacity (Logistics) | `sync:logistics` | `SYNC_NOVACITY_LOGISTICS_ENABLED` | `true` |
| Google Drive | `sync:drive` | `SYNC_GOOGLE_DRIVE_ENABLED` | `true` |
| GPRO Consulting | `sync:gpro` | `SYNC_GPRO_ENABLED` | `true` |

## Changes

### 1. Add env variables to `.env` and `.env.example`
**Files:** `.env`, `.env.example`

Add under the existing `# Sync History` section:
```
# Source Toggles
SYNC_NOVACITY_QUALITY_ENABLED=true
SYNC_NOVACITY_PRODUCTION_ENABLED=true
SYNC_NOVACITY_LOGISTICS_ENABLED=true
SYNC_GOOGLE_DRIVE_ENABLED=true
SYNC_GPRO_ENABLED=true
```

### 2. Update `config/sync.php`
**File:** `config/sync.php`

Add source enabled keys:
```php
'sources' => [
    'novacity_quality'  => env('SYNC_NOVACITY_QUALITY_ENABLED', true),
    'novacity_production' => env('SYNC_NOVACITY_PRODUCTION_ENABLED', true),
    'novacity_logistics' => env('SYNC_NOVACITY_LOGISTICS_ENABLED', true),
    'google_drive'      => env('SYNC_GOOGLE_DRIVE_ENABLED', true),
    'gpro'              => env('SYNC_GPRO_ENABLED', true),
],
```

### 3. Add `when` guards in scheduler
**File:** `routes/console.php`

Each `Schedule::command(...)` already has a `->when(...)`. Add a second condition checking the config:

```php
Schedule::command('sync:quality')
    ->when(fn () => config('sync.sources.novacity_quality') && isSyncDue('quality_interval_seconds'))
    ->everyMinute()->name('sync-quality')->withoutOverlapping(5);
```

Same pattern for all 5 commands. This is the cleanest approach â€” the scheduler never even starts the command when disabled.

### 4. No changes to SyncService, commands, or API services
The toggle is enforced at the scheduler level. The artisan commands themselves remain unchanged â€” they can still be run manually via `php artisan sync:quality` even when the scheduler is disabled, which is useful for on-demand runs.
