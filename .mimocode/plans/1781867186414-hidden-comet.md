# Plan: Add .env Switch for Historical Data Storage

## Goal
Add a single `SYNC_HISTORY_ENABLED` env variable (default `true`) that controls ALL historical data storage across sync jobs. When `false`, sync jobs run but skip snapshot creation, upsert-based accumulation, SyncLog entries, and AuditLog entries.

## Changes

### 1. Add env variable to `.env` and `.env.example`
**Files:** `.env`, `.env.example`
- Add `SYNC_HISTORY_ENABLED=true` under the sync config section

### 2. Create `config/sync.php`
**File:** `config/sync.php` (new)
- `'history_enabled' => env('SYNC_HISTORY_ENABLED', true)`
- Follows the same pattern as `config/novacity.php`

### 3. Guard `DataSnapshotService::snapshotTable()`
**File:** `app/Services/DataSnapshotService.php`
- Check `config('sync.history_enabled')` at the top of `snapshotTable()` — return `false` early when disabled

### 4. Guard `SyncService::syncTable()`
**File:** `app/Services/SyncService.php`
- At the top of `syncTable()`, resolve `$historyEnabled = config('sync.history_enabled')`
- When `$historyEnabled` is `false`:
  - **Always truncate** tables (ignore UPSERT_CONFIGS) — lines 450-452, 612-614
  - **Skip** `AuditLog::info()` and `AuditLog::error()` calls — lines 621, 625
  - **Skip** `SyncLog::record()` calls — lines 622, 626

### 5. Guard `SyncService::syncBundlingData()`
**File:** `app/Services/SyncService.php`
- When history disabled, skip `AuditLog::warn/info` and `SyncLog::record` calls in `syncBundlingData()` — lines 291, 294-296, 300

## Verification
1. Set `SYNC_HISTORY_ENABLED=false` in `.env`
2. Run `php artisan sync:quality` (or any sync command)
3. Verify: no rows inserted into `data_snapshots`, `sync_logs`, `audit_logs`
4. Verify: UPSERT tables get truncated instead of accumulated
5. Set back to `true`, re-run, verify history is recorded again
6. Run `php artisan test` to ensure no regressions
