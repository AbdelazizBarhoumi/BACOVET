## Phase 1: Fix Calculation & Formula Errors in Backend

### 1A. Fix WIP calculation direction (F-REQ-206)
**File:** `app/Http/Controllers/Api/ProductionController.php:509`
- **Bug:** `wip => max(0, (int) ($c?->sortie ?? 0) - (int) ($e?->engagement ?? 0))` — REVERSED
- **Fix:** `wip => max(0, (int) ($e?->engagement ?? 0) - (int) ($c?->sortie ?? 0))`
- Spec: `Quantité engagement - Quantité Sortie coupe`

### 1B. Fix Coupe Coverage formula direction (F-REQ-311)
**File:** `app/Http/Controllers/Api/ProductionController.php:987`
- **Bug:** `$delta = $qteCoupee - $qteEngagee` — REVERSED
- **Fix:** `$delta = $qteEngagee - $qteCoupee`
- Spec: `(Qté lancée - Qté coupée) / Cadence hebdomadaire`

### 1C. Fix Top Operators formula (F-REQ-210)
**File:** `app/Http/Controllers/Api/ProductionController.php:429-462`
- **Bug:** Backend uses `SUM(minutes_produites) / SUM(minutes_presence)` — doesn't include `temps_operation`
- Spec: `(Qté produite indiv × Temps d'opération) / Minutes présence × 100`
- **Fix:** Join with `temps_operation` table and compute `(SUM(qte × temps_reel_s) / SUM(minutes_presence)) * 100`
- Impact: `topOperators()` method + `breakdown('efficience_operateur')` (line 542-576)

### 1D. Fix WIP breakdown hardcoded status (Rule 1 violation)
**File:** `app/Http/Controllers/Api/ProductionController.php:650`
- **Bug:** `$status = $wip <= $target ? 'green' : ($wip <= $target * 2 ? 'orange' : 'red')` — hardcoded
- **Fix:** Use `$this->kpi->wipStatus($wip, $target * 2)` (target * 2 because wipStatus uses cadence, not half-cadence)

### 1E. Fix OWE formula (F-REQ-204)
**File:** `app/Http/Controllers/Api/ProductionController.php:220`
- **Current:** `$oweValue = ($stats->avg_eff * $avgSot) / $avgSam` — conflates efficiency with SOT/SAM ratio
- Spec: `(Qté déclarée × SAM) / (Effectif × Minutes présence) × 100`
- **Fix:** Compute from `qte_produite.quantite × SAM / (effectif × minutes_presence)` per chain, then aggregate. If SOT/Effectif unavailable, keep proxy but add a `proxy: true` flag in response.

### 1F. Fix other hardcoded statuses in breakdowns (Rule 1)
**File:** `app/Http/Controllers/Api/ProductionController.php`
- Line 800: `taux_archivage` — hardcodes `$pct >= 85 ? 'green' : ($pct >= 70 ? 'orange' : 'red')`
  - **Fix:** Use `$this->kpi->efficienceStatus($pct)` or create `tauxArchivageStatus()`
- Line 839: `respect_temps_estime` — hardcodes `$pct >= 90 ? 'green' : ($pct >= 80 ? 'orange' : 'red')`
  - **Fix:** Create and use a `respectTempsEstimeStatus()` method
- Line 878: `temps_acceptes` — hardcodes `$pct >= 80 ? 'green' : ($pct >= 60 ? 'orange' : 'red')`
  - **Fix:** Create and use a `tempsAcceptesStatus()` method

### 1G. Fix N+1 query in coupe chain coverage (F-REQ-310)
**File:** `app/Http/Controllers/Api/ProductionController.php:1018`
- **Bug:** `DB::table('sync_gpro_chain_planning')->where('chaine', $ch)` inside `map()` loop
- **Fix:** Eager-load `sync_gpro_chain_planning` before the loop, key by `chaine`
