# Plan: UI/UX Enhancement — Group & Merge KPI Rows

## What
1. **Sort by KPI** — rows with the same KPI are grouped together
2. **Rowspan merge** — when consecutive rows share the same KPI and Name, those cells span multiple rows instead of repeating
3. **Add row sorting** — new rows with an existing KPI are inserted next to matching KPI rows

## File to Modify
- `resources/js/routes-v1/pages/data.tsx`

## Approach

### 1. Sort `filtered` by KPI then by name
Change the `filtered` useMemo to sort rows:
```
filtered.sort((a, b) => a.kpi.localeCompare(b.kpi) || a.name.localeCompare(b.name))
```

### 2. Compute rowSpan groups
After sorting, scan consecutive rows and compute which cells to merge:
- `kpiSpan[rowIndex]` = number of consecutive rows with same KPI
- `nameSpan[rowIndex]` = number of consecutive rows with same KPI + Name
- First row of group renders the cell with `rowSpan=N`
- Subsequent rows in group skip rendering that cell (return `null` for that `<td>`)

### 3. Render with rowSpan
- KPI column: first row of KPI group gets `rowSpan=kpiSpan`, other rows skip the `<td>`
- Name column: first row of KPI+Name group gets `rowSpan=nameSpan`, other rows skip the `<td>`
- Add visual separator (thicker border-top) between different KPI groups

### 4. Sorting on addRow
When `addRow` creates a new row, insert it next to rows with the same KPI (if any) instead of at the end. Sort after insert.

## Verification
1. Rows with same KPI (e.g. 3 rows of F-REQ-101) show KPI cell spanning 3 rows
2. Within those 3 rows, if 2 share the same name, Name cell spans 2 rows
3. Adding a new F-REQ-101 row places it next to existing F-REQ-101 rows
4. Different KPI groups have visual separation
