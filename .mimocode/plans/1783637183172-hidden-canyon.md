# Fix Formula Column Display

## Problem
The Formula column in the data mapping table shows KPI codes and names instead of variable expressions:
- Header shows "Formule F-REQ-102" (KPI code)
- Variable chips show "BR GTD" (name) instead of "Nombre de rejet suite inspection commande" (variable)
- Formula item dropdowns also show names instead of variables

## Root Cause
In `resources/js/routes-v1/pages/data.tsx`, the `FormulaBuilder` component (lines 150-240) uses `gr.name` as the primary display value instead of `gr.variable`.

## Fix

### File: `resources/js/routes-v1/pages/data.tsx`

1. **Line 178** — Remove or simplify the header that shows the KPI code:
   ```
   Before: <div className="font-semibold text-muted-foreground mb-0.5">Formule {kpi}</div>
   After:  <div className="font-semibold text-muted-foreground mb-0.5">Formule</div>
   ```

2. **Line 183** — Change variable chip display to prioritize `variable` over `name`:
   ```
   Before: {gr.name || gr.variable || `Var ${gr.id}`}
   After:  {gr.variable || gr.name || `Var ${gr.id}`}
   ```

3. **Line 197** — Change formula item label to use `variable`:
   ```
   Before: label: groupRows.find((gr) => gr.id === Number(e.target.value))?.name || ""
   After:  label: groupRows.find((gr) => gr.id === Number(e.target.value))?.variable || ""
   ```

4. **Line 201** — Change formula item dropdown display to prioritize `variable`:
   ```
   Before: {gr.name || gr.variable || `Var ${gr.id}`}
   After:  {gr.variable || gr.name || `Var ${gr.id}`}
   ```

5. **Line 237** — Change "add variable" dropdown display to prioritize `variable`:
   ```
   Before: label: groupRows.find((gr) => gr.id === Number(e.target.value))?.name || ""
   After:  label: groupRows.find((gr) => gr.id === Number(e.target.value))?.variable || ""
   ```

## Verification
1. Run `npm run build` or `npm run dev` to compile
2. Open the data mapping page in the browser
3. Verify the Formula column shows variable expressions (e.g., "Nombre de rejet suite inspection commande") instead of names (e.g., "BR GTD")
4. Verify the Formula header no longer shows the KPI code
