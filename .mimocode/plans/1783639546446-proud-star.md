# Plan: Adapt to New data.json Format

## What
The data.json has been restructured from a Postman collection to a flat array of endpoint objects. The controller must be rewritten to parse this new format.

## New data.json Structure
```json
[
  {
    "name": "01 — ItemTrxEnq (SDT)",
    "method": "GET",
    "endpoint": "http://100.76.6.178:4100/api/data/itemtrxenq?limit=100&offset=0",
    "status": 200,
    "response": {
      "success": true,
      "endpoint": "itemtrxenq",
      "columns": ["IsSplit", "SONo", ...],
      "data": [{...}, ...]
    }
  }
]
```

Each item has: `name`, `method`, `endpoint` (full URL), `status`, `response` (with `columns` array and `data` array for data endpoints).

## File to Modify
- `app/Http/Controllers/Api/NovacityEndpointsController.php` — rewrite `__invoke()` and `sample()` to parse the new flat array format

## Changes

### `__invoke()` — extract endpoints
Parse the JSON array directly (not regex). For each item:
1. Extract slug from `endpoint` URL: parse path after host, take segments after `/api/`, strip query params
2. Extract fields from `response.columns` array
3. Extract name, method from the item
4. Return the same format: `{slug: {name, method, fields}}`

### `sample($slug)` — extract sample data
Parse the JSON array directly. For each item:
1. Match slug from endpoint URL
2. Return `response` as the sample data (it contains `columns`, `data`, etc.)

## Verification
1. `GET /novacity-endpoints` returns correct endpoint map from new format
2. `GET /novacity-endpoints/sample/itemtrxenq` returns the response object with columns and data
3. The `/data` page loads endpoints and shows them in the dropdown
4. Test Exec and Preview columns work with the sample data
