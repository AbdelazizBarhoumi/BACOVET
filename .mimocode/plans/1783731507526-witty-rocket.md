### Step 4: Fix Delete Error Handling & Rollback
**File:** `resources/js/routes-v1/pages/data.tsx`

Update `remove` callback (line 1007-1015) to:
- Accept full row data or re-fetch the row from current state to allow rollback
- Add toast notifications for success/failure
- On API failure, re-insert the row at its original position

Better approach: keep a snapshot of the row before removing, and re-add on failure:
```tsx
const remove = useCallback(async (id: number) => {
  let removedRow: DataMappingRow | undefined;
  setRows((rs) => {
    const idx = rs.findIndex((r) => r.id === id);
    if (idx !== -1) removedRow = rs[idx];
    return rs.filter((r) => r.id !== id);
  });
  dirtyRef.current.delete(id);
  try {
    await deleteMapping(id);
    toast.success("Ligne supprimée");
  } catch {
    if (removedRow) {
      setRows((rs) => [...rs, removedRow!]);
      toast.error("Échec de la suppression — ligne restaurée");
    }
  }
}, []);
```
