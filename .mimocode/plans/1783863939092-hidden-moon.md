# Fix: Base URL save + use properly

## Issues found
1. **Proxy forwards `baseUrl` to Novacity** — line 15 reads all query params including `baseUrl`, then line 29 sends them all to Novacity. The `baseUrl` param gets forwarded to the Novacity API which doesn't understand it.
2. **15s timeout lost** — the `executeRow` timeout was removed in a previous edit.
3. **Save effect missing CSRF token** — the POST to `/api/settings` doesn't send the XSRF token, which may fail with 419.

## Fixes

### 1. Strip `baseUrl` from forwarded query params
**File:** `app/Http/Controllers/NovacityProxyController.php` (line 15)

Change:
```php
$query = $request->query();
```
To:
```php
$query = collect($request->query())->except('baseUrl')->all();
```

### 2. Restore 15s timeout in `executeRow`
**File:** `resources/js/lib/exec.ts` (lines 194-201)

Restore the AbortController timeout pattern.

### 3. Add XSRF token to settings POST
**File:** `resources/js/routes-v1/pages/data.tsx` (line 924-933)

Add the `getCsrfToken` helper (already exists in `dataMappingApi.ts`) or inline the cookie read.

## Verification
1. Type a URL → check `settings` table has the row
2. Refresh → field empty, env fallback used for API calls
3. Type URL again → DB updates, API calls use new URL
4. Proxy logs show correct base URL, no `baseUrl` leaked to Novacity
