# Fix: Test Column Stuck Spinning in /data/ Page

## Root Cause

In `resources/js/routes-v1/pages/data.tsx`, when a test fetch is **aborted** (because the user edits another row, triggering a re-render and cleanup), the catch block at **line 1078** does:

```ts
} catch (e) {
    if (signal?.aborted) return;       // ← returns WITHOUT clearing testLoading
    ...
}
```

This leaves `testLoading[rowId] = true` permanently. The effect re-run then skips this row because `needsRecompute` is false (the old test value still exists), so the spinner never recovers.

## Fixes (3 changes, all in `data.tsx`)

### Fix 1 — Clear `testLoading` on abort (line 1078)

**Before:**
```ts
if (signal?.aborted) return;
```

**After:**
```ts
if (signal?.aborted) {
    React.startTransition(() => setTestLoading((m) => ({ ...m, [r.id]: false })));
    return;
}
```

This is the primary fix. When a fetch is aborted, we must clear the loading state before returning.

### Fix 2 — Clear `testLoading` on endpoint change (line 1777)

In the `onEndpointChange` handler, `testValues` and `directErrors` are cleared but `testLoading` is not. Add:

```ts
setTestLoading((m) => ({ ...m, [r.id]: false }));
```

Right after the existing `setTestValues` and `setDirectErrors` clears (~line 1786).

### Fix 3 — Include stuck-loading rows in recompute targets (line 1030)

Add `testLoading[r.id]` as a condition so rows stuck in loading state get re-processed:

**Before:**
```ts
const needsRecompute = !testValues[r.id] || aggSignature !== prevSignature || testValues[r.id] === "Erreur" || testValues[r.id] === "Pas de sample";
```

**After:**
```ts
const needsRecompute = !testValues[r.id] || testLoading[r.id] || aggSignature !== prevSignature || testValues[r.id] === "Erreur" || testValues[r.id] === "Pas de sample";
```

This is a safety net — if `testLoading` is somehow still true, the row will be re-fetched.
