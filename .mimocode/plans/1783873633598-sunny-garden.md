# Audit Log for /data/ Page — Implementation Plan

## Goal
Add a field-level audit log with pagination to the `/data/` page. Every change to `data_mappings` (create, update, delete, batch) is recorded with: **who** changed it, **what field** changed, **old value**, and **new value**. Displayed as a paginated table below the main mapping table. Seed/reset operations are NOT audited (bulk initial data, not user edits).

---

## Step 1 — Database Migration

**File**: `database/migrations/2026_07_12_000000_create_data_mapping_audit_logs_table.php` (new)

```php
Schema::create('data_mapping_audit_logs', function (Blueprint $table) {
    $table->id();
    $table->foreignId('user_id')->nullable()->constrained()->nullOnDelete();
    $table->unsignedBigInteger('data_mapping_id')->nullable(); // null after delete
    $table->string('kpi', 50);             // always preserved, even after delete
    $table->string('action', 20);          // created | updated | deleted
    $table->string('field', 50);           // column name that changed
    $table->text('old_value')->nullable();  // JSON-encoded
    $table->text('new_value')->nullable();  // JSON-encoded
    $table->timestamps();

    $table->index('kpi');
    $table->index('data_mapping_id');
    $table->index('created_at');
});
```

Design: **one row per changed field per operation**. Editing `name` + `endpoint` = 2 rows. This gives granular old/new diffs.

{
    protected $fillable = [
        'user_id', 'data_mapping_id', 'kpi', 'action',

## Step 3 — Auditor Service

**File**: `app/Services/DataMappingAuditor.php` (new)

A service class instantiated per request with `$userId`. Methods:

- `recordCreated(DataMapping $mapping)` — logs each non-null field as `action=created`, `old_value=null`
- `recordDeleted(DataMapping $mapping)` — logs each non-null field as `action=deleted`, `new_value=null`
- `recordUpdated(DataMapping $mapping, array $oldValues, array $newValues)` — diffs `$oldValues` vs `$newValues`, writes one row per changed field with `action=updated`
- `recordBatch(array $changes)` — calls `recordUpdated` for each row in the batch

The `$fillable` fields from DataMapping are the only fields audited. Complex types (modules array, formula object) are stored as `json_encode()`.

### `update()`
```php

// After update:

```

### `destroy()`
```php
// Before delete:
$auditor = new DataMappingAuditor($request->user()?->id);
$auditor->recordDeleted($mapping);
$mapping->delete();
```

### `batchUpdate()`
Inside the existing `foreach ($validated['mappings'] as $item)` loop, add audit capture:
```php
$row = DataMapping::find($id);
if ($row) {
    $old = $row->only(DataMapping::$fillable);
    $row->update($item);
    $new = $row->only(DataMapping::$fillable);
    $auditor->recordUpdated($row, $old, $new);
}
```

### `seedFromKpiSeed()`
No audit logging — seed/reset creates bulk initial data, not user-initiated edits.

## Step 5 — API Route

**File**: `routes/web.php` (add inside `data-mappings` group)

```php
Route::get('/audit-logs', [DataMappingController::class, 'auditLogs']);
```

**Controller method** `auditLogs()`:
```php
public function auditLogs(Request $request): JsonResponse
{
    $query = DataMappingAuditLog::with('user')
        ->orderByDesc('created_at');

    // Optional filters
    if ($kpi = $request->input('kpi')) {
        $query->where('kpi', $kpi);
    }
    if ($action = $request->input('action')) {
        $query->where('action', $action);
    }

    $perPage = min((int) $request->input('per_page', 25), 100);
    $logs = $query->paginate($perPage);

    return response()->json($logs);
}
```

Response shape (Laravel paginator):
```json
{
  "data": [...],
  "current_page": 1,
  "last_page": 5,
  "per_page": 25,
  "total": 120
}
```

## Step 6 — Frontend API Service

**File**: `resources/js/services/dataMappingApi.ts` (add)

```ts
export interface AuditLogEntry {
    id: number;
    user_id: number | null;
    data_mapping_id: number | null;
    kpi: string;
    action: 'created' | 'updated' | 'deleted';
    field: string;
    old_value: string | null;
    new_value: string | null;
    created_at: string;
    user: { id: number; name: string } | null;
}

export interface PaginatedAuditLogs {
    data: AuditLogEntry[];
    current_page: number;
    last_page: number;
    per_page: number;
    total: number;
}

export const fetchAuditLogs = async (params: {
    page?: number;
    per_page?: number;
    kpi?: string;
    action?: string;
}): Promise<PaginatedAuditLogs> => {
    const qs = new URLSearchParams();
    if (params.page) qs.set('page', String(params.page));
    if (params.per_page) qs.set('per_page', String(params.per_page));
    if (params.kpi) qs.set('kpi', params.kpi);
    if (params.action) qs.set('action', params.action);
    return fetchWithToken(`${BASE_URL}/audit-logs?${qs}`);
};
```

Key design decisions:
- **Starts collapsed** — click to expand and fetch first page
- **Server-side pagination** using existing shadcn `Pagination` component (first real usage)
- **Configurable page size** via dropdown: 10 / 25 / 50 / 100 (default 25)
- **Filters**: KPI dropdown (reuse existing kpi list), Action filter (created/updated/deleted/all)
- **Value formatting**: `modules` → comma-separated tags, `formula` → compact JSON, booleans → "Oui"/"Non", null → "—"
- **Color-coded action**: created=green, updated=blue, deleted=red
- **`refreshKey`** state incremented after each save so if the section is open, it auto-refetches
