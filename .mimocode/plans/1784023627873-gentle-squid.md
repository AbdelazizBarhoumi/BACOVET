# Fix: Array values displayed as full JSON in Formula column

## Problem

When a variable's `variable_key` resolves to an array across multiple records (e.g., `Défauts contrôle RFID`), `buildExecutionResult()` in `exec.ts` returns the full array as the output string: `[640,240,236,304,728,921,...]`. This long string is stored in `testValues[id]` and displayed in:

1. **FormulaBuilder variable chips** (data.tsx:461-463) — the `={previewValues[gr.id]}` span shows the full array
2. **Test values column** (data.tsx:2035-2036) — `ResultBadge` shows `testValues[r.id]` with `max-w-[200px]` but the long text still stretches the column

## Root cause

`stringifyResult()` in `exec.ts:51-56` does `JSON.stringify(value)` for arrays, producing unbounded-length strings. No truncation is applied anywhere in the pipeline.

## Fix (2 files)

### 1. `resources/js/lib/exec.ts` — truncate array output in `stringifyResult()`

Modify `stringifyResult()` to cap array display length. When the result is an array with more than 5 elements, show `[640, 240, 236, ... (+25)]` instead of the full `[640,240,236,...]`.

```ts
export function stringifyResult(value: unknown): string {
  if (value === null || value === undefined) return "null";
  if (typeof value === "string") return value;
  if (typeof value === "number" || typeof value === "boolean") return String(value);
  if (Array.isArray(value)) {
    const MAX_DISPLAY = 5;
    if (value.length <= MAX_DISPLAY) return JSON.stringify(value);
    const first = value.slice(0, MAX_DISPLAY).join(", ");
    return `[${first}, ... +${value.length - MAX_DISPLAY}]`;
  }
  return JSON.stringify(value);
}
```

This ensures all downstream consumers (testValues display, formula preview chips, ResultBadge) get a short string without changing the computation logic.

### 2. `resources/js/routes-v1/pages/data.tsx` — truncate display in FormulaBuilder variable chips

Add a helper function `truncateDisplay` near the `computeFormula` function (around line 390) that truncates any string value to a max display length:

```ts
function truncateDisplay(val: string, max = 40): string {
  if (val.length <= max) return val;
  return val.slice(0, max) + "…";
}
```

Then update line 462 in `FormulaBuilder`:

```tsx
// Before:
<span className="text-muted-foreground font-mono text-[9px] shrink-0">={previewValues[gr.id]}</span>

// After:
<span className="text-muted-foreground font-mono text-[9px] shrink-0" title={previewValues[gr.id]}>={truncateDisplay(previewValues[gr.id])}</span>
```

This provides a second safety net: even if a non-array long value slips through, the chip won't stretch the column. The `title` attribute preserves the full value on hover.
