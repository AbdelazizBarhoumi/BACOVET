# Plan: Add Human-Readable Formula to Data Mappings Config

## Goal
Add a `formula_readable` key to each KPI in `config/data-mappings.php` that shows the formula as a human-readable string, e.g. `"Nombre des OF soldés archivés / Nombre total des OF * 100"`.

## Current Formula Structure

```php
'formula' => [
    'items' => [
        ['type' => 'variable', 'ref' => 72, 'label' => 'Nombre des OF soldés archivés'],
        ['type' => 'operator', 'op' => '/'],
        ['type' => 'variable', 'ref' => 73, 'label' => 'Nombre total des OF'],
        ['type' => 'operator', 'op' => '*'],
        ['type' => 'number', 'value' => 100],
    ],
],
```

## Target Output

```php
'formula' => [...],  // existing array kept as-is
'formula_readable' => 'Nombre des OF soldés archivés / Nombre total des OF * 100',
```

## Files to Modify

| File | Action |
|---|---|
| `app/Console/Commands/ExportDataMappings.php` | **Modify** — add `formula_readable` generation in `buildConfig()` |

## Implementation

Add a helper method `buildReadableFormula(?array $formula): ?string` that:
1. Returns `null` if formula is null or has no items
2. Iterates through `formula['items']`:
   - `type === 'variable'` → append `$item['label']`
   - `type === 'operator'` → append `$item['op']`
   - `type === 'number'` → append `$item['value']`
3. Joins all parts with spaces
4. Returns the string

Add `'formula_readable' => ...` line right after the `'formula'` line in `buildConfig()`.

## Verification

1. Remove old config: `Remove-Item config/data-mappings.php`
2. Run: `php artisan export:mappings`
3. Lint: `php -l config/data-mappings.php`
4. Verify `formula_readable` appears on KPIs that have formulas
5. Verify KPIs with `formula => null` have `formula_readable => null`
