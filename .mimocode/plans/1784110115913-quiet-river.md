# Plan: Artisan Command to Extract Page Data from data.json

## Goal
Create an Artisan command that reads `storage/app/public/data.json` (cached API responses) and `config/data-mappings.php` (KPI definitions), then extracts the actual data each page needs based on its KPIs and variables. Output a single JSON file grouped by page.

## Data Sources

1. **`config/data-mappings.php`** — defines pages → KPIs → variables (with `endpoint`, `variable_key`, `filter_key`, `filter_value`, `fn`)
2. **`storage/app/public/data.json`** — 62 cached API responses, each with `endpoint` (full URL) and `response.data[]` (rows of data)

## Endpoint Matching

- `data.json` has full URLs: `http://100.76.6.178:4100/api/data/itemtrxenq?limit=100&offset=0`
- `data-mappings.php` has relative paths: `api/data/itemtrxenq`
- Match by parsing the URL: `parse_url($url, PHP_URL_PATH)` → `ltrim($path, '/')` → strip `?query`

## Output JSON Structure

```json
{
  "quality": {
    "kpis": {
      "F-REQ-101": {
        "name": "BR",
        "variables": [
          {
            "variable": "Nombre de rejet suite inspection commande",
            "endpoint": "api/data/itemtrxenq",
            "variable_key": "ItemNo",
            "variable_type": "Complex",
            "value": 123,
            "raw_data": [...]
          }
        ]
      }
    }
  },
  "production": { ... },
  ...
}
```

For each variable:
- `value` — the extracted value after applying filter + function (e.g., first match, sum, latest)
- `raw_data` — the full `response.data[]` rows from the matching endpoint (for debugging/inspection)

## Implementation Steps

### Step 1: Create `app/Console/Commands/ExtractPageData.php`

- **Signature**: `export:page-data {--output=storage/app/public/page-data.json : Output file path}`
- **Logic**:
  1. Load `config('data-mappings')` to get pages → KPIs → variables
  2. Load and parse `storage/app/public/data.json`
  3. Build an endpoint lookup map from data.json: `slug => response.data[]`
     - For each item in data.json: extract slug via `parse_url($item['endpoint'], PHP_URL_PATH)` + `ltrim` + strip leading `/` if not starting with `api/`
  4. For each page, for each KPI, for each variable:
     - Find the matching endpoint in the lookup map
     - If found: extract `variable_key` values from `response.data[]`
     - Apply filter if `is_filtered` + `filter_key` + `filter_value` are set
     - Apply function (`fn`) if `has_function` is true (Latest=last row, Sum=sum values, etc.)
     - Store the result as `value` and the full filtered rows as `raw_data`
     - If endpoint not found: `value` = null, `raw_data` = []
  5. Write JSON to output file
  6. Display summary: pages, KPIs, variables extracted, endpoints matched

### Helper: Value Extraction Logic

```
For each variable:
  1. Get rows from endpoint lookup (response.data[])
  2. If is_filtered: rows = rows where filter_key == filter_value
  3. If variable_type == 'Complex': value = first row's variable_key (scalar)
  4. If variable_type == 'Direct':
     - Collect all row[variable_key] values
     - Apply fn:
       - 'Latest' → last value
       - 'First' → first value
       - 'Sum' → array_sum
       - 'Average' → array average
       - 'Count' → count
       - 'Min' → min
       - 'Max' → max
  5. Store value + raw_data (the filtered rows)
```
