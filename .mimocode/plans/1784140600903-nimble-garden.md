# Plan: Batch External API Calls by Endpoint Path

## Problem

The `SyncInstantEndpoints` command (and the queue-based `SyncKpiEndpointJob`) fires **one HTTP request per (endpoint, key, kpi_code) task**. When the same endpoint path is used by multiple keys/KPIs, the external API gets hit repeatedly with identical requests.

**Concrete example** from `config/endpoints.php`:
- `api/data/diva_stock` has **6 tasks** (3 keys x 2 KPIs each) → **6 HTTP calls** to the same URL
- `api/data/ofabrication` has **8 tasks** → **8 HTTP calls** to the same URL
- `api/data/q/colis_total_3var` has **7 tasks** → **7 HTTP calls**
- `api/data/q/minutes_presence` has **7 tasks** → **7 HTTP calls**

All of these are identical HTTP requests — the response data is the same, only the extracted key differs.

## Solution: Group by Endpoint Path, Fetch Once, Extract All Keys

### Approach

Modify the `sync:instant-endpoints` command (`SyncInstantEndpoints.php`) to:

1. **Group tasks by endpoint path** instead of treating each task independently
2. **Fire one HTTP request per unique endpoint path** (using `Http::pool()` for concurrency between different endpoints)
3. **After getting the response, iterate all tasks for that endpoint** and extract each key from the same response data
4. **Store all KPI results to DB** as before

This means:
- 35+ unique tasks → only ~28 unique endpoint HTTP calls (depending on deduplication in the config)
- But more importantly: for endpoints like `diva_stock` with 6 tasks, it becomes **1 call instead of 6**

### Why only change `SyncInstantEndpoints.php`

- `SyncInstantEndpoints.php` is the **hot path** — called by the "Fire All" button and the scheduler for instant endpoints
- The queue-based `dispatchByFrequency()` → `SyncKpiEndpointJob` path is already parallel (one job per task) and handles non-instant frequencies where tasks may have different refresh rates. It's less wasteful since non-instant endpoints run infrequently
- The change is self-contained in one file

## Files to Modify

### `app/Console/Commands/SyncInstantEndpoints.php`

**Current flow** (pseudocode):
```
tasks = collect all (endpoint, key, kpi) combos
responses = Http::pool(tasks)  // 1 request per task
for each task + response:
    extract key → store to DB
```

**New flow** (pseudocode):
```
tasks = collect all (endpoint, key, kpi) combos
grouped = group tasks by endpoint path
responses = Http::pool(grouped)  // 1 request per unique endpoint
for each endpoint + response:
    for each task in that endpoint's group:
        extract key from response → store to DB
```

Key changes:
1. Build a `$grouped` array: `endpointPath => [tasks...]`
2. Fire `Http::pool()` with one request per unique `$grouped` key (not per task)
3. In the response processing loop, iterate `$grouped` entries and for each endpoint's response, process all its tasks against that single response data
4. Error handling stays the same per-task — if an endpoint fails, all tasks for it get `recordError`

### No changes needed to:
- `KpiEndpointService.php` — the queue path is separate and already parallel
- `SyncKpiEndpointJob.php` — unchanged
- Frontend (`kpi-endpoints.tsx`) — no UI changes
- `config/endpoints.php` — config stays the same
- `KpiEndpointController.php` — controller stays the same

## Implementation Detail

```php
// In SyncInstantEndpoints::handle()

// 1. Build grouped structure
$grouped = [];  // endpointPath => [tasks...]
foreach ($endpoints as $endpointPath => $endpointConfig) {
    foreach ($endpointConfig['keys'] ?? [] as $keyConfig) {
        $freq = $keyConfig['refresh_frequency'] ?? $endpointConfig['refresh_frequency'] ?? 'instant';
        if ($freq !== 'instant') continue;

        foreach ($keyConfig['kpis'] ?? [] as $kpiCode) {
            $task = [
                'endpoint' => $endpointPath,
                'kpi_code' => $kpiCode,
                'variable_key' => $keyConfig['variable_key'] ?? null,
                'variable_type' => $keyConfig['variable_type'] ?? 'Direct',
                'key_config' => $keyConfig,
                'endpoint_config' => $endpointConfig,
            ];
            $grouped[$endpointPath][] = $task;
        }
    }
}

$uniqueEndpoints = array_keys($grouped);
// Total tasks = sum of array_map('count', $grouped)
// Unique HTTP calls = count($uniqueEndpoints)

// 2. Fire one HTTP per unique endpoint
$responses = Http::pool(function (Pool $pool) use ($grouped, $baseUrl, $apiKey, $timeout) {
    foreach ($grouped as $i => $tasks) {
        $endpointPath = $tasks[0]['endpoint'];
        $pool->as($i)
            ->withHeaders(['x-api-key' => $apiKey])
            ->timeout($timeout)
            ->get("{$baseUrl}/{$endpointPath}");
    }
});

// 3. Process responses — one response serves all tasks for that endpoint
$ok = 0;
$errors = 0;
foreach ($grouped as $i => $tasks) {
    $response = $responses[$i];
    $endpointPath = $tasks[0]['endpoint'];

    // Validate response once per endpoint
    $data = null;
    $endpointFailed = false;
    // ... (same error handling as before, but set $endpointFailed = true on failure)

    // If endpoint failed, error all tasks
    if ($endpointFailed) {
        foreach ($tasks as $task) {
            $this->recordError($task, $errorMsg, $syncedAt);
            $errors++;
        }
        continue;
    }

    // If endpoint succeeded, extract each task's key from the same $data
    foreach ($tasks as $task) {
        // same extraction + validation + recordSuccess as before
        $ok++;
    }
}
```
