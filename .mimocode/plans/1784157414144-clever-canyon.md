# Plan: Refactor production-confection to use kpi_data table

## Current State

The production-confection page (`resources/js/routes-v1/pages/production-confection.tsx`) currently makes 4 separate API calls:
- `fetchProductionChainInfo()` → `/production/chain-info`
- `fetchOrderTracking()` → `/production/order-tracking`
- `fetchProductionTrend()` → `/production/efficience-trend`
- `fetchProductionStoppages()` → `/production/stoppage-timeline`

These endpoints query raw database tables directly (wip_chaine, efficience_chaine, of_fabrication, etc.) on every request.

## Target State

Replace all data fetching with a single call to the `kpi_data` table, which stores pre-computed KPI values. The `data-mappings.php` file defines 20+ KPIs for `production:confection` (F-REQ-102 through F-REQ-312) that are already being synced to `kpi_data`.

## Implementation Steps

### 1. Create Backend API Endpoint

**File:** `app/Http/Controllers/Api/ProductionController.php`

Add a new method `confectionKpis(Request $request)` that:
- Queries `kpi_data` table for all KPIs mapped to `production:confection` module
- Groups results by `kpi_code`
- Returns computed values from `computed_data` field (which contains the formula result)
- Returns response_data for raw values needed by the page

**New Route in `routes/web.php`:**
```php
Route::get('/confection-kpis', [ProductionController::class, 'confectionKpis']);
```

### 2. Add Frontend API Service Function

**File:** `resources/js/services/productionApi.ts`

Add new type and function:
```typescript
export type ConfectionKpiData = {
  kpi_code: string;
  name: string;
  value: number | null;
  raw_values: Record<string, unknown>;
  target: { operator: string; value: number | null; is_percentage: boolean };
  last_synced_at: string | null;
};

export const fetchConfectionKpis = (filters?: Record<string, string>) =>
  apiGet<{ data: ConfectionKpiData[]; synced_at: string | null }>('/confection-kpis', filters);
```

### 4. Set Up 1-Minute Refetch

The page already uses `useLiveData` hook with `refreshIntervalSec`. Update the interval to 60 seconds:

```typescript
useEffect(() => {
  fetchData();
  const interval = setInterval(fetchData, 60_000); // 1 minute
  return () => clearInterval(interval);
}, [fetchData]);
```

### 5. Handle Missing/Null Data

For KPIs that don't have values in `kpi_data` yet (endpoint is null or not synced), show "—" or "N/A" in the UI. The `last_synced_at` field helps identify stale data.
