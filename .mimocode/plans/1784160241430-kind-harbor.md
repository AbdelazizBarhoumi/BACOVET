# Fix Complex Variable Processing — has_function flag

## Change

In `v2Kpis()`, Complex variables should:
- If `has_function` is **false/null**: return the **whole raw_data array** as the variable value (for charts)
- If `has_function` is **true**: apply the function (Sum, Latest, etc.) to get a scalar

### File: `app/Http/Controllers/Api/ProductionController.php`

Change the Complex variable processing block (around line 1566):

```php
if ($isComplex && is_array($raw) && !empty($raw)) {
    $hasFn = !empty($varDef['has_function']);
    if ($hasFn) {
        // Apply function to get scalar
        $fn = $varDef['fn'] ?? 'Latest';
        $processed = $this->applyFunction($raw, $varKey, $fn);
        $variableValues[$varKey] = $processed;
    } else {
        // No function — collect all values for this key as an array
        $allValues = array_column($raw, $varKey);
        $variableValues[$varKey] = $allValues;
    }
    $rawData = array_merge($rawData, $raw);
    // ... filter options ...
}
```

The `computeFormula()` must also handle array values — when a variable is an array, take the first element for formula computation.

## Verification

- KPIs with `has_function: false` should show full raw_data in charts
- KPIs with `has_function: true` should show computed scalar value
- Formulas should still compute correctly using first element of arrays
