# Plan: Replace fetchSampleData calls with allEndpointData reads

## Goal
All TraceBtn/preview buttons and validation logic currently make individual network calls via `fetchSampleData(slug)`. The data is already loaded once on mount via `fetchAllSamples()` into `allEndpointData`. This plan replaces those 8 network calls with instant reads from the pre-loaded state.

## Files to modify

1. **`resources/js/routes-v1/pages/data.tsx`** — all 8 call sites + new helper + prop changes

## Changes

### 1. Add `getRecordsFromAllData` helper (~after line 944, inside component)

```ts
const getRecordsFromAllData = useCallback(
  (slug: string): Record<string, unknown>[] => allEndpointData[slug]?.response ?? [],
  [allEndpointData]
);
```

### 2. Add `allEndpointData` prop to EndpointSelector

Add to props interface (line 755):
```ts
allEndpointData: Record<string, AllEndpointRecord>;
```

Pass at call site (line 1860):
```tsx
allEndpointData={allEndpointData}
```

### 4. Simplify auto-fetch test values useEffect (line 1115-1174)

Since the 8 calls become synchronous, the auto-fetch batch loop can be simplified:
- Remove `fetchInBatches` wrapper — iterate `targets` directly with a `for` loop
- Remove `AbortController`/`signal` (no async I/O to abort)
- Remove `fetchingRef` tracking (sync, no overlapping calls)
- Keep `React.startTransition` for state updates

The `fetchInBatches` function (line 44) stays because `fetchPreview` (line 1106) still uses it for live API calls.

### 5. Remove loading states for TraceBtns

Since reads are instant, the `setTraceLoading`/`try/finally` patterns around each TraceBtn click become unnecessary:
- Line 1899-1917: Remove `setTraceLoading` in Endpoint `onTrace` (synchronous)
- Line 2016-2033: Remove `setTraceLoading` in Variable Key TraceBtn
- Line 2078-2101: Remove `setTraceLoading` in Filter TraceBtn

The `traceLoading` prop to `EndpointSelector` and `ResultBadge` can stay (renders instantly, no flicker).

### 6. Remove unused imports (line 20, 27)

Remove from data.tsx:
- `fetchSampleData` (line 20)
- `extractRecords` (line 27)

`fetchSampleData` stays exported from `dataMappingApi.ts` (other pages may use it). `extractRecords` stays exported from `lib/exec.ts`.
