# Plan: Public SQL Export API for Data Mappings Table

## Goal

Add a **public** (no auth) API endpoint `GET /data-mappings/export-sql` that returns the entire `data_mappings` table as a downloadable `.sql` file containing MySQL-compatible `UPDATE` statements.

## Files to Modify

| File | Change |
|---|---|
| `routes/web.php` | Add 1 public route (outside auth middleware) |
| `app/Http/Controllers/Api/DataMappingController.php` | Add `exportSql()` method + SQL generation helpers (~50 lines) |
| `resources/js/services/dataMappingApi.ts` | Add `exportSql()` download function (~12 lines) |

## Step 1: Route (`routes/web.php`)

Add after line 214 (closing of the `auth:data_users` group), before the Novacity section:

```php
Route::get('/data-mappings/export-sql', [DataMappingController::class, 'exportSql']);
```

No middleware — public endpoint as requested.

## Step 2: Controller Method (`DataMappingController.php`)

Add `exportSql()` public method + 3 private helpers:

**`exportSql()`** — queries all rows, builds SQL string, returns as file download:
```php
public function exportSql()
{
    $mappings = DataMapping::orderBy('kpi')->orderBy('id')->get();
    $sql = $this->buildExportSql($mappings);

    return response()->streamDownload(function () use ($sql) {
        echo $sql;
    }, 'data_mappings_export.sql', [
        'Content-Type' => 'application/sql',
    ]);
}
```

**`buildExportSql()`** — iterates rows, emits one `UPDATE` per row:
- Header: comment block with timestamp + row count
- `SET NAMES utf8mb4;`
- Each row: `UPDATE data_mappings SET col=val, ... WHERE kpi='...' AND variable='...';`
- Natural key for WHERE: `kpi` + `variable` (not `id`, since IDs differ between environments)

**`sqlValue()`** — type-aware formatting:
- `null` → `NULL`
- JSON columns (`modules`, `formula`, `graph_types`) → `json_encode()` wrapped in quotes
- Boolean columns (`is_filtered`, `has_function`, `cible_is_percentage`) → `1` / `0`
- Numeric (`cible_value`) → bare number
- Everything else → string with `addslashes()` escaping

**Columns exported** (all except `id`, `user_id`, `created_at`, `updated_at`):
`name, endpoint, variable_type, variable_key, is_filtered, filter_key, filter_value, has_function, fn, modules, formula, highlight_color, cible_operator, cible_value, cible_is_percentage, refresh_frequency, graph_types`

## Step 3: Frontend (`dataMappingApi.ts`)

Add at bottom of file:
```typescript
export const exportSql = async (): Promise<void> => {
    const response = await fetch(`${BASE_URL}/export-sql`);
    if (!response.ok) {
        throw new Error(`Export failed: ${response.status}`);
    }
    const blob = await response.blob();
    const url = window.URL.createObjectURL(blob);
    const a = document.createElement('a');
    a.href = url;
    a.download = 'data_mappings_export.sql';
    document.body.appendChild(a);
    a.click();
    document.body.removeChild(a);
    window.URL.revokeObjectURL(url);
};
```

Uses raw `fetch` (not `fetchWithToken`) since it's a public endpoint with no auth.

## Step 4: Testing

1. `curl http://localhost:8000/data-mappings/export-sql` — verify HTTP 200, correct headers, valid SQL output
2. Inspect downloaded file: header comments, `SET NAMES utf8mb4`, UPDATE statements
3. Optionally run the SQL against a test MySQL DB to verify rows update correctly
