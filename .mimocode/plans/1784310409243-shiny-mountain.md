# Plan: Superadmin SQL Sync Button on /data Page

## Overview
Add a superadmin-only button to the /data page that fetches a SQL dump from an external API endpoint, executes it against the local `data_mappings` table, then runs artisan export commands and cache clear.

## Changes

### 1. Add env variable `SQL_DUMP_URL` (`.env` / `.env.example`)
- Add `SQL_DUMP_URL=` env variable for the external API URL that returns the SQL file
- Add a config value in `config/services.php` or use `env()` directly in the controller

### 2. New API endpoint: `DataMappingController::syncFromSql`
**File**: `app\Http\Controllers\Api\DataMappingController.php`
- Add method `syncFromSql(Request $request)`
- Fetches SQL content from the URL defined in `SQL_DUMP_URL` env
- Executes the raw SQL against the database using `DB::unprepared()`
- After SQL execution, runs these artisan commands via `Artisan::call()`:
  - `export:mappings`
  - `export:endpoints`
  - `optimize:clear`
- Returns JSON with success/error and output from each command

### 3. Register the route
**File**: `routes/web.php`
- Add `POST /data-mappings/sync-sql` route inside the existing `auth:data_users` middleware group
- Add a check that the authenticated user's role is `it` (superadmin)

### 4. Add frontend API function
**File**: `resources/js/services/dataMappingApi.ts`
- Add `syncDataFromSql()` function that calls `POST /data-mappings/sync-sql`

### 5. Add the button + confirmation dialog to the data page
**File**: `resources/js/routes-v1/pages/data.tsx`
- Add a "Sync SQL" button in the toolbar, visible only when `isSuperadmin` is true
- Button triggers a confirmation dialog (similar to the existing Reset dialog)
- On confirmation, calls `syncDataFromSql()`, shows loading state, then toast with result
- Button placed near the existing superadmin Reset button

## Key Files
- `app\Http\Controllers\Api\DataMappingController.php` — new `syncFromSql` method
- `routes/web.php` — new route registration
- `resources/js/services/dataMappingApi.ts` — new API function
- `resources/js/routes-v1/pages/data.tsx` — new button + confirmation dialog

## Superadmin Detection (existing pattern)
- Frontend: `userRole === "it"` (from localStorage `data_auth_user`)
- Backend: `$request->user()->role === 'it'` (DataUser model, role is a plain string column)

3. Log in as a non-superadmin user — verify the button is hidden
