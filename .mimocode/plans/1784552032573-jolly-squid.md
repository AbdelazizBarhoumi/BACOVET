# Fix: F-REQ-207 (arrêts non planifiés) shows meaningless 275647.0 as Big Number

## Root Cause

F-REQ-207 has `graph_types: ['Chronologie (Timeline)', 'Big Number avec couleur']` in `config/data-mappings.php:2849`.

The v2 production page (`resources/js/routes-v2/pages/production.tsx:90-108`) splits KPIs into `bigNumbers` (those with `'Big Number avec couleur'`) and `chartKpis`. Since F-REQ-207 has that type, it renders as a `ProductionKpiCard`.

The value shown (275647.0) comes from the `v2Kpis()` method which, for F-REQ-207, applies `fn: 'Sum'` to the `LostTime` variable — summing ALL lost time minutes across all records. This is meaningless as a single number.

On the `/data` page, it correctly shows `_` because that page doesn't render big numbers for this KPI type.

**F-REQ-207 is a timeline/chronology KPI — it should NOT appear as a Big Number card.**

## Fix

Remove `'Big Number avec couleur'` from F-REQ-207's graph_types so it only renders as a Chronologie (Timeline) chart.

### Files to modify

| File | Line | Change |
|------|------|--------|
| `config/data-mappings.php` | 2849 | `['Chronologie (Timeline)', 'Big Number avec couleur']` → `['Chronologie (Timeline)']` |
| `database/seeders/DataMappingGraphTypesSeeder.php` | 39 | Same change for consistency |

### Verification

1. Hit `/production/v2-kpis?module=production:confection` — F-REQ-207 should still have `graph_types: ["Chronologie (Timeline)"]` and NOT include `"Big Number avec couleur"`
2. On the v2/production page, F-REQ-207 should render only as a bar chart timeline, not as a big number card
3. The `/data` page should continue showing `_` (no change expected)
