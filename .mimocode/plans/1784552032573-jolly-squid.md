# Plan: Fix F-REQ-211 — Shows "Array (32)" Instead of a Number

## Problem

F-REQ-211 card shows `Array (32)` instead of a numeric value.

## Root Cause

Two issues in `config/data-mappings.php` line 3079-3102:

1. **`has_function: false`** → engine returns array of 32 objects instead of a scalar
2. **Formula `ref: 67`** → broken reference (only 1 variable, should be `ref: 0`)

## Expected Behavior

- **No filter**: show the **latest** TempsStandard value (last element)
- **With ProdGroup filter**: show the single TempsStandard for that chain

## Fix

Change in `config/data-mappings.php` line 3090:

```php
// OLD:
'has_function' => false,

// NEW:
'has_function' => true,
'fn' => 'Latest',
```

And fix formula ref at line 3098:

```php
// OLD:
'ref' => 67,

// NEW:
'ref' => 0,
```

### File to modify

| File | Change |
|------|--------|
| `config/data-mappings.php` | Set `has_function: true`, `fn: "Latest"`, fix `ref: 0` |

## Verification

1. Card shows a number (latest TempsStandard) instead of "Array (32)"
2. Filter by ProdGroup → card shows the single TempsStandard for that chain
