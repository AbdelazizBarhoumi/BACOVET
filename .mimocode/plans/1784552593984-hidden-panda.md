# Fix: Cible column dropdown shows `<`/`>` instead of `<=`/`>=`

## Problem
In the `/data/` page, the Cible column's operator dropdown uses HTML entities (`&lt;=`, `&gt;=`) for display text. When the user selects `<=` or `>=`, the value is stored correctly in the database, but the dropdown trigger displays only `<` or `>` (the `=` is dropped).

**Root cause**: `LightDropdown`'s `selectedLabel` (line 71-80 in `LightDropdown.tsx`) extracts the display label by reading `child.props.children` from each `DataSelectItem`. JSX text content using HTML entities like `&lt;=` — where an entity is immediately followed by additional characters — results in the `children` prop being only the entity-decoded portion (`<`), dropping the trailing `=`.

## Fix

**File**: `resources/js/routes-v1/pages/data.tsx` (lines 2113-2117)

Replace HTML entities in the Cible dropdown `DataSelectItem` children with JavaScript string expressions:

```tsx
// Before:
<DataSelectItem value="<">&lt;</DataSelectItem>
<DataSelectItem value=">">&gt;</DataSelectItem>
<DataSelectItem value=">=">&gt;=</DataSelectItem>
<DataSelectItem value="<=">&lt;=</DataSelectItem>
<DataSelectItem value="=">>=</DataSelectItem>

// After:
<DataSelectItem value="<">{"<"}</DataSelectItem>
<DataSelectItem value="">{">"}</DataSelectItem>
<DataSelectItem value=">=">{">="}</DataSelectItem>
<DataSelectItem value="<=">{"<="}</DataSelectItem>
<DataSelectItem value="=">{"="}</DataSelectItem>
```

This is the only file that needs to change. JavaScript string expressions bypass JSX entity parsing entirely, ensuring the full text (including `=`) is preserved in `child.props.children`.
