## The Bug

`flushDirty()` in `data.tsx` (line ~1243-1263) builds the batch save payload but **omits** `extra_filters`, `chart_config`, and `graph_types`. These fields are updated in local state via `updateImmediate` but never persisted to the database.

## Fix

**File:** `resources/js/routes-v1/pages/data.tsx` — function `flushDirty`

Add to the payload object:
```typescript
graph_types: row.graph_types ?? null,
chart_config: row.chart_config ?? null,
extra_filters: row.extra_filters ?? null,
```
