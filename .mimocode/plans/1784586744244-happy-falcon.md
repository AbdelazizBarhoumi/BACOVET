# Fix: modal_formula_readable_not_passing_through

## Problem

The V2 KPI Detail Modal always shows "Valeur brute" in the formula section, even though `formula_readable` exists in the data-mappings config for every KPI.

## Root Cause

`formula_readable` exists in `KpiConfig` (line 26 of `kpiFilterEngine.ts`) but is **not** in the `ComputedKpi` interface (lines 77-94) and is **not** passed through in the `computeKpi()` return (lines 599-616). The modal receives a `ComputedKpi` cast as `any`, so `kpi.formula_readable` is always `undefined`.

## Fix (2 changes in 1 file)

**File:** `resources/js/lib/kpiFilterEngine.ts`

1. Add `formula_readable: string | null;` to the `ComputedKpi` interface (after line 85)
2. Add `formula_readable: kpi.formula_readable,` to the `computeKpi()` return object (after line 607)

## Verification

- Open `/v2/production`, click any KPI card
- Modal should show the formula text instead of "Valeur brute"
- If a KPI genuinely has no `formula_readable` in config, it should still show "Valeur brute"
