# Fix: "Used Endpoints" Missing from V2 Production Modal

## Problem

The V2 production page modal (`V2KpiDetailModal.tsx`) renders `kpi.endpoints` (line 210), but this field is **always `undefined`** at runtime because:

1. `KpiConfig` and `ComputedKpi` types lack an `endpoints` field
2. `toKpiConfigs()` extracts `endpoint` per variable but never aggregates them into a top-level `endpoints: string[]`
3. `computeKpi()` doesn't pass endpoints to the `ComputedKpi` result
4. The unsafe `as unknown as V2KpiItem` cast at `production.tsx:193` hides the type mismatch

## Fix (4 files, ~10 lines)

### 1. Add `endpoints` to types — `resources/js/lib/kpiFilterEngine.ts`

- Add `endpoints?: string[];` to `KpiConfig` interface (after line 44)
- Add `endpoints?: string[];` to `ComputedKpi` interface (after line 95)

### 2. Collect endpoints in `toKpiConfigs()` — `resources/js/hooks/useV2KpiCalc.ts`

In the `.map()` callback (line 13-42), after building `variables`, collect unique non-null endpoints:

```ts
const endpoints = [...new Set(variables.map(v => v.endpoint).filter(Boolean))] as string[];
```

Add `endpoints` to the returned object.

### 3. Pass endpoints through in `computeKpi()` — `resources/js/lib/kpiFilterEngine.ts`

In both the fast path (line 387-405) and slow path return, add:

```ts
endpoints: kpi.endpoints ?? null,
```

### 4. Remove unsafe cast — `resources/js/routes-v2/pages/production.tsx`

Change line 193 from `selectedKpi as unknown as V2KpiItem | null` to `selectedKpi as V2KpiItem | null` (or better, ensure `ComputedKpi` is compatible with what `V2KpiDetailModal` expects).
