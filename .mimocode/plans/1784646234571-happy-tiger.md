# Fix: Extra Filters dropdown should show dynamic common keys from KPI endpoints

## Problem
The `ExtraFiltersEditor` dropdown in the V1 data page (`resources/js/routes-v1/pages/data.tsx`) shows a hardcoded list of 9 static column keys (`COMMON_COLUMN_KEYS` on line 236-239) instead of dynamically computing the common JSON keys from the actual KPI endpoint response data.

## Root Cause
`ExtraFiltersEditor` (lines 158-234) renders checkboxes from a compile-time constant array. It does not receive or use any data from the endpoint responses.

## Solution
Compute the intersection of JSON keys across all endpoint responses in a KPI group, and pass those as the dropdown options to `ExtraFiltersEditor`.

### Data flow
1. `allEndpointData` (state, line 1057) already holds `Record<string, AllEndpointRecord>` loaded via `fetchAllSamples()` — each entry has a `response: Record<string, unknown>[]` array of sample records.
2. For each KPI group (rows `filtered.slice(i, i + ks)`), collect all unique `endpoint` values from the group's rows.
3. For each endpoint, get the first record from `allEndpointData[slug].response` and extract its keys.
4. Compute the intersection of all key sets → these are the "shared column keys" for that KPI group.
5. Pass this array to `ExtraFiltersEditor` as a new `sharedKeys` prop.

### Files to modify

**1. `resources/js/routes-v1/pages/data.tsx`**

- **`ExtraFiltersEditor` component (lines 158-234)**: Add a `sharedKeys: string[]` prop. Replace `COMMON_COLUMN_KEYS` references with `sharedKeys`. Remove the hardcoded `COMMON_COLUMN_KEYS` constant (lines 236-239).

- **Table rendering (around line 2316-2322)**: When `isFirstInKpi` is true, compute `sharedKeys` from the group's endpoints using `allEndpointData` (already in scope via closure from `DataMappingPage`). Pass it to `<ExtraFiltersEditor sharedKeys={sharedKeys} .../>`.

#### Change 4: Compute shared keys per KPI group in the table

At the `ExtraFiltersEditor` usage site (lines 2316-2322), compute the shared keys:

```tsx
{isFirstInKpi ? (() => {
  const groupRows = filtered.slice(i, i + ks);
  // Collect unique endpoints from this KPI group
  const groupEndpoints = [...new Set(groupRows.map(gr => gr.endpoint).filter(Boolean) as string[])];
  // Compute intersection of JSON keys from all endpoint responses
  const keySets = groupEndpoints.map(ep => {
    const rec = allEndpointData[ep];
    if (!rec || !rec.response || rec.response.length === 0) return [];
    return Object.keys(rec.response[0]);
  });
  const sharedKeys = keySets.length > 0
    ? keySets[0].filter(k => keySets.every(ks => ks.includes(k)))
    : [];
  return (
    <td rowSpan={ks} className="px-2 py-1.5 min-w-[200px]">
      <ExtraFiltersEditor
        value={r.extra_filters}
        onChange={(v) => updateImmediate(r.id, { extra_filters: v })}
        sharedKeys={sharedKeys}
      />
    </td>
  );
})() : null}
```

### Edge cases
- **Single endpoint in group**: `sharedKeys` = all keys from that endpoint's first record. This is correct — a single endpoint has no "intersection" constraint, so all its keys are shared.
- **No endpoint or empty response**: `sharedKeys` = `[]`. The custom input section still allows manual entry.
- **Rows without endpoints**: Filtered out via `filter(Boolean)`.
