# Fix widget style application — styles on inner card, not outer div

## Problem
When applying border radius, border color, background, etc. via the inspector, styles are applied to the **outer wrapper div** of each widget, not the **visible card/content inside**. This causes:
- **KPI**: Double border — outer div has configurable border, inner `Card` has hardcoded `rounded-lg border border-2`. Increasing radius only rounds the outer div, not the visible card.
- **Chart widgets**: `wrap()` has both `border` and `bg-card` CSS classes AND `boxStyle` inline styles — they conflict. The CSS class wins over inline styles for some properties.
- **All wrap() widgets**: The `border` class forces `border-width: 1px` regardless of config.

## Root Cause
1. **`wrap()` function** (widget-renderer.tsx:54-61): Has `className="... border border-border bg-card"` which conflicts with `boxStyle()` inline styles. CSS classes and inline styles fight each other.
2. **KPI widget** (widget-renderer.tsx:68-79): Renders an outer `<div style={boxStyle}>` then `<KpiCard>` which wraps in `<Card className="rounded-lg border border-2 bg-card p-3">` — hardcoded border/radius on the inner card.

## Fix

### File: `resources/js/components/builder/widget-renderer.tsx`

#### 1. Fix `wrap()` — remove conflicting CSS classes
Change the wrap div to use ONLY inline styles from `boxStyle()`, no conflicting CSS classes:

```tsx
const wrap = (child: React.ReactNode) => (
  <div
    className="h-full w-full flex flex-col"
    style={{
      ...style,
      borderWidth: c.borderWidth,
      borderStyle: c.borderStyle || (c.borderWidth ? "solid" : undefined),
      borderColor: c.borderColor,
      border: c.showBorder === false ? "none" : undefined,
    }}
  >
    {c.showLabel !== false && c.label && (
      <div className="text-[10px] uppercase tracking-widest text-muted-foreground mb-1 shrink-0">{c.label}</div>
    )}
    <div className="flex-1 min-h-0">{child}</div>
  </div>
);
```

Actually, simpler — just remove the `border border-border bg-card` classes since `boxStyle` already handles `background`, `borderColor`, `borderWidth`, `borderStyle`:

```tsx
const wrap = (child: React.ReactNode) => (
  <div className={`h-full w-full flex flex-col ${c.showBorder === false ? "border-0" : ""}`} style={style}>
    {c.showLabel !== false && c.label && (
      <div className="text-[10px] uppercase tracking-widest text-muted-foreground mb-1 shrink-0">{c.label}</div>
    )}
    <div className="flex-1 min-h-0">{child}</div>
  </div>
);
```

#### 2. Fix KPI widget — remove outer div, pass styles to KpiCard
The KPI widget bypasses `wrap()` and renders its own outer div with `boxStyle`. The inner `KpiCard` wraps in `<Card>` which has hardcoded `rounded-lg border border-2 bg-card p-3`.

Fix: Remove the outer div's style application. Instead, pass config styles as a `className`/`style` prop to `KpiCard` → `Card`.

But since `Card` is in `v1/primitives.tsx` (shared component), we can't easily change its interface. Instead, wrap the KpiCard in a div that applies ONLY the boxStyle (no double border):

```tsx
case "kpi": {
  const v = c.mockValue ?? 0;
  const t = c.target ?? 0;
  const status = t ? (v >= t ? "ok" : v >= t * 0.9 ? "warn" : "bad") : "ok";
  return (
    <div className="h-full w-full" style={{ ...style, border: "none" }}>
      <KpiCard ... />
    </div>
  );
}
```

Wait, this still has the problem that `Card` inside `KpiCard` has its own hardcoded border. The real fix is to make `Card` accept custom styles.

**Better approach**: Since `Card` already accepts `className` and `...props`, we can pass `style` to it via `KpiCard`. But `KpiCard` doesn't forward props to `Card`.

**Simplest fix**: Don't use `KpiCard`'s internal `Card`. Instead, render the KPI content directly with the boxStyle applied. This means inlining the KPI rendering.

Actually, the simplest and cleanest fix is:
1. Remove the `border border-border bg-card` classes from `wrap()` (boxStyle already handles these)
2. For KPI, apply `style` to the div and remove the hardcoded border from the inner Card by passing a className override

Since `Card` accepts `className` prop and merges it: `className={rounded-lg border border-2 bg-card p-3 ${className}}`, we can pass `className="border-0 rounded-none bg-transparent p-0"` to reset Card's styles, then let the outer div's boxStyle handle everything.

But KpiCard doesn't forward className to Card. So the fix is to **not use KpiCard** and instead render the KPI content directly with boxStyle.

### Revised approach — inline KPI rendering

Replace the KPI case to render directly without KpiCard's Card wrapper:

```tsx
case "kpi": {
  const v = c.mockValue ?? 0;
  const tgt = c.target ?? 0;
  const status = tgt ? (v >= tgt ? "ok" : v >= tgt * 0.9 ? "warn" : "bad") : "ok";
  const color = STATUS_COLOR[status];
  return (
    <div className="h-full w-full flex flex-col p-3" style={style}>
      {c.showLabel !== false && c.label && (
        <div className="text-[10px] uppercase tracking-widest text-muted-foreground mb-1">{c.label}</div>
      )}
      <div className="text-[10px] text-muted-foreground">{c.kpiCode}</div>
      <div className="flex items-end justify-between mt-1">
        <div className="text-3xl font-black leading-none" style={{ color }}>
          {v.toFixed(c.decimals ?? 1).replace(".", ",")}
          <span className="text-base ml-1 font-bold text-muted-foreground">{c.unit ?? ""}</span>
        </div>
      </div>
      <div className="flex items-center justify-between mt-2">
        {c.showTarget && tgt ? <div className="text-[10px] text-muted-foreground">Objectif : {tgt}</div> : null}
        {status === "ok" ? <Check className="h-4 w-4 text-green-500" /> : status === "bad" ? <AlertTriangle className="h-4 w-4 text-red-400" /> : null}
      </div>
      {c.showSparkline && c.mockSeries && (
        <div className="h-8 mt-1">
          <ResponsiveContainer>
            <AreaChart data={c.mockSeries.map((s, i) => ({ x: i, y: s.v }))}>
              <Area type="monotone" dataKey="y" stroke={color} fill={color} fillOpacity={0.2} strokeWidth={1.5} />
            </AreaChart>
          </ResponsiveContainer>
        </div>
      )}
    </div>
  );
}
```

This way, the boxStyle (borderRadius, borderColor, bg, etc.) applies directly to the visible KPI card div — no double border.
