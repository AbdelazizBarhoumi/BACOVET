# V3 Page Builder — Full CRUD Backend Persistence

## Goal
Replace localStorage-only persistence with a MySQL backend so pages (registry + widget layouts) survive browser data clears, work across devices, and are shareable.

## Overview of Changes

### 1. Migration: `builder_pages` table
**File**: `database/migrations/2026_07_22_000001_create_builder_pages_table.php`

```php
Schema::create('builder_pages', function (Blueprint $table) {
    $table->id();
    $table->string('slug')->unique();
    $table->string('name');
    $table->json('layout')->nullable();   // stores {version, widgets} — the full PageLayout
    $table->timestamps();
});
```

- `slug` — unique, used in URLs (`/p/{slug}`)
- `name` — display name
- `layout` — JSON blob holding `{version: 1, widgets: Widget[]}` (same structure as current localStorage `bacovet.builder.{slug}`)

### 3. API Controller: `BuilderPageController`
**File**: `app/Http/Controllers/Api/BuilderPageController.php`

Endpoints (all return JSON):

| Method | URI | Action | Description |
|--------|-----|--------|-------------|
| `GET` | `/api/builder-pages` | `index()` | List all pages (id, slug, name, created_at, updated_at) |
| `GET` | `/api/builder-pages/{slug}` | `show($slug)` | Get one page with full layout |
| `POST` | `/api/builder-pages` | `store()` | Create page (validates: `name` required, `slug` auto-generated if missing) |
| `PUT` | `/api/builder-pages/{id}` | `update($id)` | Update page name/slug/layout |
| `DELETE` | `/api/builder-pages/{id}` | `destroy($id)` | Delete page |
| `POST` | `/api/builder-pages/{id}/duplicate` | `duplicate($id)` | Duplicate page + layout |

The `store` method generates a unique slug server-side (same slugify logic as frontend). The `update` method for slug changes returns the new slug so the frontend can redirect.

### 4. API Routes
**File**: `routes/web.php` (add inside the existing authenticated group or as a new public group)

Since `/v3` and `/p/{slug}` are currently public (no auth), the API routes should also be public to match. Add near line 47:

```php
// ── V3 BUILDER API ─────────────────────────────────────────────────
Route::prefix('api/builder-pages')->group(function () {
    Route::get('/', [BuilderPageController::class, 'index']);
    Route::get('/{slug}', [BuilderPageController::class, 'show']);
    Route::post('/', [BuilderPageController::class, 'store']);
    Route::put('/{id}', [BuilderPageController::class, 'update']);
    Route::delete('/{id}', [BuilderPageController::class, 'destroy']);
    Route::post('/{id}/duplicate', [BuilderPageController::class, 'duplicate']);
});
```

### 5. Frontend: Replace localStorage in `pages-registry.ts`
**File**: `resources/js/lib/pages-registry.ts`

**Change**: Convert all functions from synchronous localStorage reads to async API calls.

- `readRegistry()` → `fetchPages()` (async, calls `GET /api/builder-pages`)
- `createPage(name)` → `POST /api/builder-pages`
- `renamePage(id, name)` → `PUT /api/builder-pages/{id}`
- `changeSlug(id, slug)` → `PUT /api/builder-pages/{id}` with slug change
- `duplicatePage(id)` → `POST /api/builder-pages/{id}/duplicate`
- `deletePage(id)` → `DELETE /api/builder-pages/{id}`
- `getPageBySlug(slug)` → `GET /api/builder-pages/{slug}`

The `usePagesRegistry()` hook will fetch from the API on mount. The slugify function moves to shared use (backend generates slug, frontend keeps a copy for preview).

Remove: `LAYOUT_KEY`, `REG_KEY`, all `localStorage.getItem/setItem/removeItem` calls.

Keep: The `slugify()` function (still used for client-side preview/validation).

---

### 6. Frontend: Update builder `store.tsx` save/load
**File**: `resources/js/components/builder/store.tsx`

**Change**: The `save()` and reset logic in the builder context.

- `save()` (line 177-181): Change from `localStorage.setItem` to `PUT /api/builder-pages/{pageId}` with `{layout: {version: 1, widgets}}`
- `useEffect` load (line 98-113): Load layout from props (passed via Inertia) instead of localStorage
- `reset()` (line 183-190): Call `PUT /api/builder-pages/{pageId}` with `{layout: null}` to clear the layout on server

Remove: `storageKey()` function, all `localStorage` references.

---

### 7. Frontend: Update page renderer `[slug].tsx`
**File**: `resources/js/pages/v3/p/[slug].tsx`

**Change**: The server already passes `slug` as a prop. Extend the Inertia route to also pass the full page data (name + layout) as props, eliminating the need for a client-side fetch to find the page.

Update the route in `web.php`:
```php
Route::get('/p/{slug}', function ($slug) {
    $page = \App\Models\BuilderPage::where('slug', $slug)->firstOrFail();
    return Inertia::render('v3/p/[slug]', [
        'slug' => $page->slug,
        'pageName' => $page->name,
        'layout' => $page->layout['widgets'] ?? [],
    ]);
})->name('v3.page');
```

The `[slug].tsx` component receives `pageName` and `layout` as props, passes `layout` as `defaultLayout` to `DashboardBuilder`.
