# Fix: Builder KPI API Returns Empty — Parameter Mismatch

## Root Cause

`GET /api/builder-kpis/data?codes=F-REQ-104,F-REQ-201`

The frontend sends codes as a **comma-separated string** in the URL. The controller does `$request->input('codes')` which returns `"F-REQ-104,F-REQ-201"` (a string). Then the check `!is_array($codes)` is true, so it returns `[]` immediately. **Every KPI request returns empty.**

## Fix

**File:** `app/Http/Controllers/Api/BuilderKpiController.php`

Update the `data()` method to handle comma-separated strings:
```php
$codes = $request->input('codes');
if (is_string($codes)) {
    $codes = array_map('trim', explode(',', $codes));
}
if (empty($codes) || !is_array($codes)) {
    return response()->json([]);
}
```

This is the only change needed. No frontend changes required.
