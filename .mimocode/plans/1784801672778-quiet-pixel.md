# Plan: Create BuilderPageSeeder with KPI Widgets

## Goal
Create a database seeder that populates the `builder_pages` table with one page per top-level key from `config/data-mappings.php`, and includes KPI widgets in each page's layout so they're visible when the page loads.

## Database Schema
`builder_pages` table — `slug`, `name`, `layout` (JSON), timestamps.

Layout JSON format (from `types.ts`):
```json
{
  "version": 1,
  "widgets": [
    { "id": "abc123", "type": "kpi", "x": 0, "y": 0, "w": 3, "h": 3,
      "config": { "kpiCode": "F-REQ-101", "label": "BR", "unit": "%", "decimals": 1, "target": 5, "accent": "#22c55e", "showTarget": true, "showLabel": true } }
  ]
}
```

## Layout Strategy

Arrange widgets in a responsive grid across 24 columns:
- Place a `text` header widget (page title) at y=0, full width
- Arrange KPI cards in rows of 8 (3w × 8 = 24)
- Charts (6–8w) placed 3 per row or 1 per row for large combos
- After each row, advance y by max h in that row + 1

## Implementation

### 1. Modify `database/seeders/BuilderPageSeeder.php`

Load `config/data-mappings.php`, iterate each page key, extract unique KPIs (deduplicated by code), map `graph_types[0]` to a WidgetType, build the widget array with grid positions, and save as the `layout` JSON.

Each widget gets:
- `id`: random 7-char string (matching `uid()` pattern from types.ts)
- `type`: mapped from graph_types
- `x`, `y`: calculated grid position
- `w`, `h`: from DEFAULT_SIZE
- `config.kpiCode`: the KPI code (e.g. `F-REQ-101`)
- `config.label`: the KPI name
- `config.unit`: `%` if target is percentage
- `config.decimals`: 1
- `config.target`: target value
- `config.accent`: `#22c55e`
- `config.showTarget`: true
- `config.showLabel`: true
