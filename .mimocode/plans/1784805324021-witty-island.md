# Fix: Table Grid Excel Features Not Working

## Root Causes Found

### Bug 1: Cursor never set on cell click (CRITICAL)
**File:** `canvas.tsx:241-250`

`handleCellSelect` updates `tableSel` but NEVER sets `tableCursor`. The keyboard handler (line 56) checks `const cur = tableCursor[selectedId]` and bails out if `cur` is falsy. So arrow keys, Tab, Enter, and Escape do nothing after clicking a cell.

**Fix:** In `handleCellSelect`, also set `tableCursor` to the clicked cell position.

### Bug 2: Context menu copy only copies 1 cell
**File:** `canvas.tsx:306-308`

The `onCopy` callback from the context menu calls `copyCells(tg, r, c, r, c)` — copying only the single right-clicked cell. Should copy the full selection range instead.

**Fix:** Change to copy the bounding box of `tableSel[w.id]`, falling back to the single cell.

### Bug 3: Unused import
**File:** `table-grid.tsx:1`

`GripVertical` is imported but not used (the drag handle was removed during development but the import stayed). Causes lint warning.

**Fix:** Remove the unused import.

### Bug 4: Arrow key handler uses stale `tg` after table mutation
**File:** `canvas.tsx:45-176`

The `tg` variable is captured at effect setup time. After a paste or delete mutates the table, the handler still references the old `tg` until React re-renders and the effect re-runs. This can cause off-by-one errors on rapid key presses.

**Fix:** Read `widget.config.tableGrid` fresh inside the handler instead of using the captured `tg`.

## Detailed Fixes

### Fix 1: canvas.tsx handleCellSelect — set cursor on click

```ts
const handleCellSelect = (widgetId: string, r: number, c: number, add: boolean) => {
  setTableCursor((prev) => ({ ...prev, [widgetId]: [r, c] }));  // ADD THIS
  setTableSel((prev) => {
    const key = `${r},${c}`;
    const cur = prev[widgetId] ?? [];
    if (add) {
      return { ...prev, [widgetId]: cur.includes(key) ? cur.filter((k) => k !== key) : [...cur, key] };
    }
    return { ...prev, [widgetId]: [key] };
  });
};
```

### Fix 2: canvas.tsx onCopy — use full selection range

Replace the onCopy callback:
```ts
onCopy={(r, c) => {
  const tg = w.config.tableGrid;
  if (!tg) return;
  const sel = tableSel[w.id] ?? [];
  if (sel.length > 0) {
    const parsed = sel.map((k) => k.split(",").map(Number));
    const r1 = Math.min(...parsed.map((p) => p[0]));
    const r2 = Math.max(...parsed.map((p) => p[0]));
    const c1 = Math.min(...parsed.map((p) => p[1]));
    const c2 = Math.max(...parsed.map((p) => p[1]));
    setTableClipboard((p) => ({ ...p, [w.id]: copyCells(tg, r1, c1, r2, c2) }));
  } else {
    setTableClipboard((p) => ({ ...p, [w.id]: copyCells(tg, r, c, r, c) }));
  }
}}
```

### Fix 3: canvas.tsx keyboard handler — read tg fresh

Replace `const tg = widget.config.tableGrid;` (captured at setup) with a fresh read inside the handler:
```ts
const handler = (e: KeyboardEvent) => {
  const target = e.target as HTMLElement;
  if (target.tagName === "INPUT" || target.tagName === "TEXTAREA" || target.tagName === "SELECT") return;

  const widget = widgets.find((w) => w.id === selectedId);
  if (!widget || widget.type !== "table-grid" || !widget.config.tableGrid) return;
  const tg = widget.config.tableGrid;  // FRESH read each keystroke

  const cur = tableCursor[selectedId];
  const sel = tableSel[selectedId] ?? [];
  // ... rest of handler
};
```

This also means the outer `if` guard and `tg` variable should move inside the handler.

### Fix 4: table-grid.tsx — remove unused import

Remove line 1: `import { GripVertical } from "lucide-react";`
