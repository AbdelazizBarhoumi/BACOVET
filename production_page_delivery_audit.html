🔴 BR Bundling (F-REQ-106) — Novacity endpoints INACTIVE
The KPI card for BR Bundling exists and is wired correctly in the MD, but the Novacity API endpoints it depends on — rejets_suite_inspection_paquet_jour_en_cours (§3.20) and inspections_paquet_jour_en_cours (§3.21) — are both marked État: Inactif. BR Bundling will always return null / zero at runtime.
Fix → Ask Novacity admin to activate §3.20 and §3.21, OR add a null-guard in kpis() that renders a "données non disponibles" grey card instead of a broken value.
🔴 WIP Flux chart (F-REQ-206) — qte_engagement missing date column
The MD's wip() function groups qte_engagement by date over 30 days. But Novacity's qte_engagement (§3.15) returns only commande · of · article · quantite_engagee — no date field. The area chart cannot be rendered. The engagement axis will always be empty.
Fix → Request Novacity to add a date column to qte_engagement (§3.15), or derive date from linked of_fabrication.DtDebut via a join in the backend query.
🔴 Inline vs Endline chart (Row 9 Coupe) — quantity columns absent
Novacity's vwInlineVSEndlineComparison (§2.7) exposes only LOGDATE · ShiftCode · SHORTNAME · OPERA — no quantity, no count, no defect measure. The stacked bar chart in the MD requires a numeric Y-axis per operation per chain. The chart cannot render anything meaningful from these 4 columns.
Fix → Request Novacity to extend §2.7 with at least one quantity column (e.g. Qty or DefectCount), or map this chart to vwDefect (§2.9) which already has Qty + OpNo + ProdGroup.
🔴 GPRO Consulting data (SAM / SOT / effectif / BPD / EPD / EHD / cadence / objectif) — not in Novacity API
Chain card (Row 1) and coverage formulas (F-REQ-310, 311) require fields from sync_gpro_article_master, sync_gpro_chain_planning, and sync_gpro_of_dates. None of these tables are exposed in the Novacity API (neither §2 nor §3). The MD assumes they are synced to the dashboard's central DB by a separate mechanism — but this mechanism is unverified and undocumented.
Fix → Document and test the GPRO Consulting sync job. Confirm sync_gpro_* tables are populated before delivery, with fallback null-handling in chainInfo() using the metadata.missing_fields response key already defined in the MD.
🟡 BR Print (F-REQ-108) — Google Drive sync unverified
BR Print data source is Google Drive (as per CDC), not Novacity. The MD references a sync_drive_br_print table that must be populated by a separate Drive connector. This sync is not documented in either PDF and its status is unknown. If it fails silently, the BR Print KPI card will show stale or null data without any user-visible error.
Fix → Verify Drive → DB sync is active. Add a synced_at freshness check: if last sync > 6h ago (BR Print refreshes 4x/day), show a stale-data warning badge on the KPI card.
🟡 F-REQ-312 (Objectif par chaîne) — embedded only, not a KPI card
The CDC defines F-REQ-312 as a standalone "Big Number avec couleur" KPI. In the MD, the daily objective appears only as a small field (QTÉ: 500) inside the chain card (Row 1) — not as a dedicated KPI card. The Annexe 1 lists it for both Confection and Coupe.
Consider: If the client has signed off on the chain card layout, the embedded display may be acceptable. If not, add a KPI card (·312) to Row 2 alongside the existing cards.
🟡 F-REQ-217 (Taux fiabilité données) — table only, no Gauge KPI
The main CDC body (§3.3) defines this as a Gauge Chart with 95% target. The MD implements it only as a tagging table in Row 9 (Coupe tab). However, Annexe 1 assigns F-REQ-217 to the Méthodes view (priority 2), not to the production view — so this omission may be intentional.
Confirm: if the Méthodes view is a separate page, no change needed here. If it must also appear in Production, add a gauge KPI card (·217) to Row 2 for the Coupe tab.
